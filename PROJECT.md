# Лексикон ЧГК — словарный запас авторов вопросов

Корпусное исследование лексического разнообразия **авторов вопросов ЧГК**, по
аналогии с исследованием рэперов Ильи Иноземцева (rap.inozemtsev.lol) и нашим
[stand-up research](../standup_research/PROJECT.md).

## Цель
По базе вопросов «Что? Где? Когда?» посчитать для каждого автора метрику
«у кого богаче словарь» и выложить интерактивную инфографику (один HTML-файл,
данные вшиты, поиск + сортировка).

## Данные
- Источник: дамп базы вопросов ЧГК (gotquestions / db.chgk.info), формат — один
  JSON на пакет/турнир: `~/Downloads/Quiz Packs/packs/*.json` (6701 пакет).
- Структура: pack -> `tours[]` -> `questions[]`. У вопроса есть `text`, `comment`,
  `answer`, `source` и список `authors` (`id`, `name`, `gender`) и `editors`.
- Масштаб: **436 454 вопроса, 9 430 авторов**.

## Методология (1:1 с Иноземцевым/стендапом, + наши правки)
- Корпус автора = `text` + `comment` всех его вопросов, в хронологическом порядке
  (по `endDate`/`pubDate` вопроса, затем по id).
- Только **кириллические** токены (латиница/цифры/имена собственные латиницей
  выбрасываются). Токенизация — просто по кириллице; лемматизация — `pymorphy3`.
- Метрика: число **уникальных лемм в первых 25 000 словах** корпуса (как у
  Иноземцева). Корпус < 25 000 слов = «бледный круг» (метрика на меньшем окне).
- Дедуп по глобальному `id` вопроса (один вопрос бывает в нескольких пакетах).
- Со-авторские вопросы засчитываются **каждому** из перечисленных авторов.
- Порог отображения: 10 000 слов корпуса (359 авторов).

## Пайплайн (`scripts/`)
- `build_report.py` — два прохода по всем пакетам (pass1 = подсчёт слов и отбор
  отображаемых, pass2 = сбор токенов) + лемматизация первых 25k -> `report.json`
  (метрика словаря + `pid` профиля gotquestions).
- `compute_features.py` — тяжёлые «фишки»: weighted log-odds (фирменные слова:
  темы + стиль), родственные авторы (косинус tf-idf), граф соавторства, слово/имя/гео
  года, штампы/клише. Кэширует агрегаты в `scripts/_raw.pkl` (пересчёт: `--fresh`).
  Пишет: дополняет `report.json` + `coauthors.json`, `years.json`, `cliches.json`.
- `build_site.py` — собирает 5 self-contained страниц из JSON (общая тема/навигация):
  `index.html` (рейтинг + раскрытие: слова-маркеры/темы/родственники), `method.html`
  (объяснение метода), `years.html` (слово года), `coauthors.html` (canvas-граф),
  `cliches.html` (штампы).

Порядок: `build_report.py` -> `compute_features.py` -> `build_site.py`.
Метод фирменных слов: weighted log-odds с приором Дирихле (Monroe et al. 2008).

- Медиана словаря у надёжных: **5908** уникальных лемм.
